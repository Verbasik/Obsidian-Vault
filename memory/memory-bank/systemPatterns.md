# System Patterns

## Архитектурные принципы
- Единый источник истины: `memory/memory-bank/` + `memory/rules/memory-bank.mdc`.
- Приоритеты: P0 > P1 > P2. Конфликты решаются по приоритетам.
- Триггеры обновлений: фичи, архитектура, требования, статус, паттерны, явный запрос.

## Паттерны процесса
- Active Context: отражает текущий фокус, всегда актуален (P0).
- Progress Ledger: фиксирует статус работ с чекбоксами и стрелками состояний.
- Decision/Knowledge Log: журнал памяти (`memory-bank.mdc`) для нетривиальных решений и уроков.
- STaR Loop: INITIAL → ANALYZE → IMPROVE → VALIDATE (качество ≥0.8).

## Паттерны знаний (учебные)
- Zettelkasten Atomic Notes: одна концепция — одна заметка; ссылки в обе стороны.
- MOC (Map of Content): обзорные узлы тем; стандартизированное имя `00-MOC <тема>.md`.
- Zettel→Card Pipeline: внизу заметки — блок «Карточки» с Q/A; периодическая выгрузка в `Anki/anki_cards.csv`.
- Coverage Map: чек‑листы тем по разделам с уровнями глубины; обновляется по мере создания заметок.
- Paper Summaries: заметки по статьям (arXiv) со связями к базовым концептам.

## Потоки
- Инициализация → Сбор вводных → Ведение работ → Обновления по триггерам → Отчет.

## Конвенции
- Файловая организация: `Subject/{Anki, Zettelkasten, Infographics, *.pdf}`.
- Нейминг MOC: `00-MOC <ТЕМА>.md` (как в текущем хранилище).
- Карточки: CSV c явными полями (`Front,Back,Tags,Difficulty,Source`).
